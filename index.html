The problem

Every agent session starts from zero

Coding agents are only as good as what they've been told — and right now, what they've been told dies with the session.

Without Repo Brain

  • Dev A's agent decides to use Zod over Joi for validation — that decision lives only in Dev A's local session.
  • Dev B's agent, on a different machine, re-derives the same decision from scratch, or picks something else entirely.
  • Conventions drift silently across a team as each agent reasons in isolation.
  • Institutional knowledge about *why* a call was made evaporates the moment the terminal closes.

With Repo Brain

  • Dev A's decision is written into a knowledge graph scoped to the repo, not the laptop.
  • Dev B's agent recalls it automatically on the very first relevant prompt — no manual sync step.
  • Conventions stay consistent because every agent is reasoning from the same shared memory.
  • When a file is deleted, the memory tied to it is pruned — the graph tracks the codebase's actual current state.
How it works

One pipeline, two Claude Code hooks

No commands to run mid-session. It happens on every prompt and every response, automatically.

1

A developer types a prompt

Claude Code's UserPromptSubmit event fires before the model sees anything.

2

Repo Brain recalls relevant memory

recall-context.mjs searches the repo's Cognee dataset for anything related to the prompt, and injects the result as additionalContext — before Claude answers.

3

Claude answers with full context

The agent responds as if it already knew the team's prior decision — because now it does.

4

The turn is remembered

On Stop, remember-decision.mjs compresses what just happened into one decision-sentence and writes it into the same shared dataset.

5

Any other clone of the repo inherits it

Because the dataset name is derived from git remote origin url, every laptop on the same repo converges on the exact same graph — automatically, with no manual sync.

The Cognee lifecycle

Four verbs, fully wired

Repo Brain doesn't just store text — it uses Cognee's whole memory lifecycle.

01

remember()

Every agent decision gets summarized and written into the repo's dataset via Cognee's add + cognify pipeline.

02

recall()

Every new prompt is checked against the graph before Claude answers, via Cognee's graph-grounded search.

03

improve()

When a human overrides an agent's suggestion, that feedback re-weights the graph so the next answer is sharper.

04

forget()

A git post-commit hook detects deleted files and prunes the memory tied to them, so the graph never goes stale.

Architecture

What's actually in the repo

Hover any file to see what it does.

.claude/settings.json
Wires the two hooks into Claude Code's UserPromptSubmit and Stop lifecycle events. Claude Code picks this up automatically — nothing to run.
.claude/hooks/recall-context.mjs
Fires before Claude answers. Searches the shared dataset and injects results as context. Fails open — a broken API call never blocks a prompt.
.claude/hooks/remember-decision.mjs
Fires after Claude finishes responding. Compresses the turn into one decision-sentence and writes it into the shared dataset.
lib/cognee-client.mjs
The wrapper around Cognee's REST API — remember, recall, improve, and forget, all in one place.
lib/repo-id.mjs
Hashes git remote origin url into a single dataset name. This is the entire trick behind cross-laptop sharing.
scripts/forget-deleted-files.mjs + post-commit.sh
A git hook that detects deleted files after every commit and prunes their memory from the graph.
Setup

From zero to shared memory

Every teammate does this once. After that, it's invisible.

1Get your Cognee Cloud connection details

In your Cognee Cloud dashboard, generate an API key, and copy the tenant-specific base URL and Tenant ID shown on the "Connection Details" panel.

2Clone the repo and set your environment
git clone https://github.com/Mohan-0001/Repo-Brain.git
cd Repo-Brain
cp .env.example .env
# edit .env with COGNEE_BASE_URL, COGNEE_TENANT_ID, COGNEE_API_KEY
3Install the forget() git hook
cp scripts/post-commit.sh .git/hooks/post-commit
chmod +x .git/hooks/post-commit
4Verify the hooks standalone
echo '{"last_assistant_message":"Decided to use Zod for validation.","cwd":"."}' \
  | node .claude/hooks/remember-decision.mjs

echo '{"prompt":"what validation library should I use?","cwd":"."}' \
  | node .claude/hooks/recall-context.mjs

The second command should print back the decision from the first. That round-trip working is the entire proof the system is wired correctly.

5Open Claude Code — that's it
claude

The hooks fire silently on every prompt and every response from here on. No command to "start recording."
